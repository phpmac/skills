# 推荐奖励体系 (直推/级差/团队奖)

## A. 直推奖

A邀请B, B交易/质押时A按固定比例获奖. 链上 `mapping(address => address) inviter` 记录关系, 绑定后不可修改.

## B. 级差奖

多层级推荐, 逐级递减比例(如L1:10%, L2:5%, L3:3%), 向上遍历inviter链分发. 必须设 MAX_LEVEL 防Gas超限.

## C. 团队奖

按团队总业绩划分等级, 达标后获额外奖励比例. 等级判定必须同时要求自身投入 + 团队业绩.

## 安全审查清单

- [ ] inviter 绑定后不可篡改, 防循环引用 (A->B->A)
- [ ] 级差遍历有 MAX_LEVEL 上限, 各层比例之和 < 总手续费
- [ ] 有最低投入门槛防 create2 批量刷假团队
- [ ] 团队业绩防闪电贷 (冷却期/快照机制)
- [ ] unstake 时团队业绩向上正确扣减
- [ ] address(0) 的奖励去向明确 (归国库)
- [ ] owner 无法修改推荐关系
