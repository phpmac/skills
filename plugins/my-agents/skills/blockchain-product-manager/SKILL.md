# 区块链产品设计

协助整理产品文档, 审查业务逻辑漏洞, 计算代币经济/费率参数, 输出结构化的产品设计文档.

## PRD 文档模板

```markdown
# [产品/功能名称] PRD

## 1. 概述
- 产品定位: [一句话描述]
- 目标用户: [用户画像]
- 支持链: [ETH/BSC/Solana/...]
- 核心价值: [解决什么问题]

## 2. 功能清单
| 功能模块 | 优先级 | 描述 | 涉及合约 |
|---------|--------|------|---------|

## 3. 用户流程
[ASCII 流程图描述核心用户路径]

## 4. 数据模型
- 链上数据: [合约状态]
- 链下数据: [后端/数据库存储]

## 5. 合约交互设计
| 操作 | 合约方法 | 参数 | 权限 | Gas 预估 |
|------|---------|------|------|---------|

## 6. 经济模型
[代币分配/费率/激励机制]

## 7. 风险评估
[业务逻辑风险/合约风险/监管风险]
```

## 手续费计算框架

```
总手续费 = 交易金额 * 费率
平台收入 = 总手续费 * 平台分成比例
LP收入 = 总手续费 * LP分成比例
回购销毁 = 总手续费 * 销毁比例

// 验证: 各比例之和必须 = 100%
```

## 代币分配计算

```
总供应量 = X
团队 = X * 团队比例% (需锁仓计划)
投资者 = X * 投资比例% (需释放计划)
社区/空投 = X * 社区比例%
流动性 = X * 流动性比例%

// 验证: 各比例之和必须 = 100%
// 建议: 团队+投资者 <= 30%, 社区相关 >= 50%
```

## 质押收益计算

```
APR = (年奖励总量 * 代币价格) / (总质押量 * 代币价格) * 100%
APY = (1 + APR/n)^n - 1  // n=复利次数
```

## 逻辑漏洞审查

产品设计审查时, 使用 `contract-audit` 技能进行专业漏洞审计, 使用 `smart-contract-vuln` agent 查阅具体漏洞模式.

产品经理角度快速检查:
- 资金是否有无限授权风险
- 存取款流程是否存在重入攻击路径
- 多签/时间锁是否覆盖关键资金操作
- 通胀/通缩模型是否可持续
- 套利空间是否被恶意利用 (闪电贷攻击路径)
- 用户操作顺序是否存在 race condition
- 权限模型是否存在越权路径

## 输出规范

1. 所有数值必须附带计算过程
2. 所有比例必须验证总和 = 100%
3. 资金相关流程必须画出 ASCII 流向图
4. 权限模型必须列出角色矩阵
5. 风险项必须标注严重等级: CRITICAL / HIGH / MEDIUM / LOW
6. 参数建议必须给出范围和理由
