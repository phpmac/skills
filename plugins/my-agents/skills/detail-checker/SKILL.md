# 代码细节检查

修改或编写代码时, 结合内部逻辑和关联部分进行全面的代码检查.

## 检查清单

### 1. 上下游影响
- 修改函数时追溯上游调用方和下游依赖, 评估影响范围
- 全局搜索同类实现, 确认是否需要统一修改

### 2. 状态与副作用
- 修改状态变量后, 检查其他依赖该变量的逻辑是否受影响
- 关注边界条件: 0值/空值/最大值/溢出

### 3. 接口兼容
- 函数签名变更是否影响调用方
- 返回值类型变化是否导致下游解析失败

### 4. 测试覆盖
- 修改后现有测试是否仍能覆盖核心逻辑
- 新逻辑是否需要补充测试

### 5. 重复检查
- 类似逻辑在其他文件是否存在, 是否应提取公共函数
- 同一业务逻辑出现 2 次以上, 立即提取公共函数

### 6. 格式规范
- 中文内容使用英文标点
- 禁止 emoji
- 路径使用 `~` 代替硬编码
- 代码注释使用中文

### 7. 相关 skills/agents
- 检查是否有相关 skill/agent 可以辅助完成当前任务
- 根据问题类型选择合适的 agent 协助分析

### 8. 外部验证
- 不确定的 API/语法/方法, 使用 Context7 或 WebSearch 获取最新文档验证
- 涉及外部依赖/框架变更时, 主动搜索确认兼容性
- 不要凭记忆判断, 用 Context7/WebSearch 交叉验证关键信息

### 9. 协同分析
- 尽可能调用其他相关 agent 交叉验证
- 复杂问题不要独自判断, 结合多个 agent 的分析结果综合评估
