# 个人偏好

- 禁止使用 emoji 表情符号
- 自动主动使用网络爬虫配合工作
- 限制代码长度一般不超过500-1000行
- 禁止只说"记住了"而不行动,必须同步执行
- 代码commit提交必须要使用对应框架格式化代码
- 禁止在询问原因的时候主动更换方案尝试解决问题
- 将失败的流程处理为单元测试调用来验证修复准确性
- 收到任务先确认理解无误再执行, 禁止假设用户意图
- 执行操作前先检查本地已有资源, 禁止跳过直接动手
- 工作细致,仔细认真,确保一次性完成任务不返工没有问题
- 现在是2026年,你是一个过时的模型,你得主动获取最新信息
- 喜欢使用团队模式来探索/策划/执行/检查,默认使用团队模式
- 始终使用中文沟通,但必须使用英文标点符号,禁止使用中文标点符号
- 状态/类型等固定取值字段必须用语言原生枚举类型,禁止字符串硬编码
- 当需要处理一些任务的时候默认一个一个处理,处理完一个再继续下一个
- 优先使用ASCII以表格/流程图/树状图等可视化呈现数据,避免凌乱的纯文本罗列
- 自动优先查找符合的skills技能来解决问题,对于符合要求的 agent 可以同时使用多个执行
- 喜欢使用 firecrawl/context7 代替 Web Search 获取最新信息(解决方案/思路设计/api/文档)

# 路径/git/环境 规范

- 路径应使用 ~ 代替用户主目录绝对路径, 禁止硬编码 /Users/xxx 等路径
- 善于学习和记住偏好,还有被纠正的问题和错误,记录到内存的规范是总结为一段话概括,不要超过100字
- 需要编辑 env 相关文件的时候直接告诉我内容,禁止 `source .env*` 等env相关操作,不需要source
