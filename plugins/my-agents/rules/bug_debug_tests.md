# Bug/Debug/单元测试 规范

- 发现/处理bug要看有没有类似问题
- 一个要求就合并为一个单元测试函数
- 开发时主动编写单元测试覆盖核心逻辑
- 流程图中标注关键代码片段和变量状态变化
- 分析其他地方可能会存在类似问题或者连带影响
- 发现的bug得使用 claude code 和 cursor-escalator 交叉验证
- 对比测试需覆盖变更前后状态,验证设置前(正向)/设置后(反向)的行为差异
- 实在无法解决的就使用流程图呈现,结合关键代码片段由人工提供思路来解决
- 持续无法解决也将 需求+问题+试过的解决方案 提供给 cursor-escalator 来获取解决方案/思路
- 有时候问题无法解决也许是思路错了,得换个思路尝试 网络搜索爬虫/cursor-escalator/人工辅助
