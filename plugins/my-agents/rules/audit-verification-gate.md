# 安全审计交付标准

## 验证分级

| 类型 | 标记 | 可交付 |
|------|------|--------|
| 静态分析 (Slither/Mythril) | `## 未验证发现` | 否 |
| 符号执行/模拟交易 | `## 待验证` | 否 |
| 链上 cast call 验证通过 | `## 已验证 [severity]` + 实际返回值 | 是 |
| forge test PoC 通过 | `## 确认可利用` + 测试输出 | 是 |

## 硬规则

1. **禁止编造 PoC 数字** — 所有数值必须来自实际链上调用或测试输出
2. **禁止报告未验证的 severity** — 不得使用 "Critical" / "High"
3. **必须区分**: `## 未验证发现` vs `## 确认可利用`
4. **无法验证时**: 标注为 `## 无法验证, 疑似风险`, 提供验证步骤让用户自行决定

**关联**: `security_audit.md` 已定义此规范, 此文件为强化执行层.
