## 规范

- 如果对话中没有可蒸馏内容, 告知用户暂无
- 禁止直接写入, 必须先展示完整内容经用户确认
- 蒸馏是方法论, 不是固定答案; 提炼决策标准而非操作流程

## 蒸馏优先级

1. **纠正/踩坑** -> Rule 或 CLAUDE.md (最高优先, 防止再犯)
2. **可复用工作流** -> Skill
3. **全局行为规范** -> CLAUDE.md 追加

## 蒸馏流程

1. **回顾对话** - 重点关注: 纠正/提醒/踩坑/反复出现的问题 + 可复用的工作流
2. **提炼核心** - 纠正类: 提炼为规则 (做什么/不做什么/为什么); 工作流类: 提炼为方法论
3. **选择载体** - 根据优先级确定存放位置
4. **展示预览** - 展示完整内容 + 蒸馏来源 + 建议存放位置
5. **用户确认** - 确认后写入, 需修改则调整后重新展示
6. **写入** - 写入确定的位置

## 载体与存放

- **Rule** (`.claude/rules/<name>.md`): 纠正/踩坑/检查项, 简洁条目式, 可选 `paths` 限定范围
- **Skill** (`.claude/skills/<name>/SKILL.md`): 可复用工作流, frontmatter (name + description) + 方法论 (< 500 行)
- **CLAUDE.md** (追加, 不删除): 全局行为规范, `##` 分段, 与现有风格一致
