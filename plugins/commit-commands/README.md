# Commit Commands 插件

使用简单的命令简化 git 工作流, 包括提交, 推送和创建 PR.

## 概述

Commit Commands 插件自动化常见的 git 操作, 减少上下文切换和手动命令执行. 使用单个斜杠命令即可处理整个工作流, 而无需运行多个 git 命令.

## 命令

### `/commit`

根据暂存和未暂存的变更自动生成提交信息并创建 git commit.

**功能:**
1. 分析当前 git 状态
2. 审查暂存和未暂存的变更
3. 检查最近的提交信息以匹配仓库风格
4. 起草合适的提交信息
5. 暂存相关文件
6. 创建提交

**用法:**
```bash
/commit
```

**示例工作流:**
```bash
# 修改代码
# 然后运行:
/commit

# Claude 将会:
# - 审查您的变更
# - 暂存文件
# - 使用合适的信息创建提交
# - 显示提交状态
```

**特性:**
- 自动起草匹配仓库风格的提交信息
- 遵循约定式提交实践
- 避免提交包含密钥的文件 (.env, credentials.json)
- 在提交信息中包含 Claude Code 署名

### `/commit-push-pr`

完整工作流命令, 一步完成提交, 推送和创建 PR.

**功能:**
1. 创建新分支 (如果当前在 main 上)
2. 使用合适的信息暂存并提交变更
3. 推送分支到 origin
4. 使用 `gh pr create` 创建 PR
5. 提供 PR URL

**用法:**
```bash
/commit-push-pr
```

**示例工作流:**
```bash
# 修改代码
# 然后运行:
/commit-push-pr

# Claude 将会:
# - 创建功能分支 (如需要)
# - 提交您的变更
# - 推送到远程
# - 使用摘要和测试计划打开 PR
# - 提供 PR URL 供您审查
```

**特性:**
- 分析分支中的所有提交 (不仅是最新提交)
- 创建包含以下内容的全面 PR 描述:
  - 变更摘要 (1-3 个要点)
  - 测试计划检查清单
  - Claude Code 署名
- 自动处理分支创建
- 使用 GitHub CLI (`gh`) 创建 PR

**要求:**
- GitHub CLI (`gh`) 必须已安装并认证
- 仓库必须有名为 `origin` 的远程

### `/clean_gone`

清理已从远程仓库删除的本地分支.

**功能:**
1. 列出所有本地分支以识别 [gone] 状态
2. 识别并删除与 [gone] 分支关联的 worktree
3. 删除所有标记为 [gone] 的分支
4. 提供已删除分支的反馈

**用法:**
```bash
/clean_gone
```

**示例工作流:**
```bash
# PR 合并并删除远程分支后
/clean_gone

# Claude 将会:
# - 查找所有标记为 [gone] 的分支
# - 删除关联的 worktree
# - 删除过时的本地分支
# - 报告清理结果
```

**特性:**
- 同时处理普通分支和 worktree 分支
- 在删除分支前安全删除 worktree
- 显示清晰的删除反馈
- 如果无需清理则报告

**何时使用:**
- 合并并删除远程分支后
- 本地分支列表被过时分支占满时
- 定期仓库维护时

## 安装

此插件包含在 Claude Code 仓库中. 使用 Claude Code 时命令自动可用.

## 最佳实践

### 使用 `/commit`
- 提交前审查暂存的变更
- 让 Claude 分析您的变更并匹配仓库的提交风格
- 信任自动化信息, 但验证其准确性
- 用于开发过程中的常规提交

### 使用 `/commit-push-pr`
- 准备好创建 PR 时使用
- 确保所有变更已完成并测试
- Claude 将分析完整分支历史用于 PR 描述
- 审查 PR 描述, 必要时编辑
- 想要最小化上下文切换时使用

### 使用 `/clean_gone`
- 定期运行以保持分支列表清洁
- 合并多个 PR 后特别有用
- 安全运行 - 只删除远程已删除的分支
- 帮助维护整洁的本地仓库

## 工作流集成

### 快速提交工作流:
```bash
# 编写代码
/commit
# 继续开发
```

### 功能分支工作流:
```bash
# 跨多个提交开发功能
/commit  # 第一次提交
# 更多变更
/commit  # 第二次提交
# 准备创建 PR
/commit-push-pr
```

### 维护工作流:
```bash
# 多个 PR 合并后
/clean_gone
# 干净的工作区准备下一个功能
```

## 要求

- Git 必须已安装并配置
- 对于 `/commit-push-pr`: GitHub CLI (`gh`) 必须已安装并认证
- 仓库必须是具有远程的 git 仓库

## 故障排除

### `/commit` 创建空提交

**问题**: 没有变更可提交

**解决方案**:
- 确保您有未暂存或已暂存的变更
- 运行 `git status` 验证变更存在

### `/commit-push-pr` 无法创建 PR

**问题**: `gh pr create` 命令失败

**解决方案**:
- 安装 GitHub CLI: `brew install gh` (macOS) 或参见 [GitHub CLI installation](https://cli.github.com/)
- 认证: `gh auth login`
- 确保仓库具有 GitHub 远程

### `/clean_gone` 未找到分支

**问题**: 没有标记为 [gone] 的分支

**解决方案**:
- 运行 `git fetch --prune` 更新远程跟踪
- 分支必须从远程删除才会显示为 [gone]

## 技巧

- **与其他工具结合**: 开发时使用 `/commit`, 准备好时使用 `/commit-push-pr`
- **让 Claude 起草信息**: 提交信息分析会学习您仓库的风格
- **定期清理**: 每周运行 `/clean_gone` 维护干净的分支列表
- **推送前审查**: 始终在推送前审查提交信息和变更
